Lesson IV

Python Tuples

Immutable Ordered Collections

What is a Tuple?

  • An ordered collection
  • Immutable (cannot be changed after creation)
  • Allows duplicate values
  • Can store mixed data types

It looks like this:

my_tuple = (1, 2, 3)

Creating Tuples

Basic Syntax

numbers = (10, 20, 30)

Tuple Packing

numbers = 10, 20, 30

Single-Element Tuple

single = (5,)   # Comma is required!

Parentheses Are Not What Make a Tuple

The comma, not the parentheses, creates a tuple. 


a = (1, 2, 3)
b = 1, 2, 3

print(type(a))  # tuple
print(type(b))  # tuple

Parentheses for Clarity

Even though parentheses are optional, they are recommended for:

  • Readability
  • Preventing ambiguity
  • Multi-line formatting

coordinates = (
    40.7128,
    -74.0060
)

The Single-Element Tuple Rule

Common mistake:


single = (5)
print(type(single))  # int

Correct:


single = (5,)
print(type(single))  # tuple

Why the Comma Matters


x = 5,
print(x)        # (5,)
print(type(x))  # tuple

Parentheses group expressions. The comma creates the tuple.

Empty Tuples


empty = ()

Invalid syntax:


empty = ,   # SyntaxError

Multi-Line Tuple Creation


config = (
    "localhost",
    5432,
    True,
    30.5
)

Common in configuration and structured constants.

Tuple Creation from Iterables


numbers = tuple([1, 2, 3])
letters = tuple("abc")

print(numbers)  # (1, 2, 3)
print(letters)  # ('a', 'b', 'c')

Trailing Commas


values = (1, 2, 3,)
  • Cleaner diffs in version control
  • Easier element additions
  • Valid and recommended in long tuples

Nested Tuple Creation


data = (
    ("Alice", 25),
    ("Bob", 30),
    ("Charlie", 35)
)

Useful for structured immutable records.

Tuple Packing in Function Returns


def get_user():
    return "Alice", 25

result = get_user()
print(result)  # ('Alice', 25)

Python automatically packs return values into a tuple.

Tuple Creation with Generators


squares = tuple(x**2 for x in range(5))
print(squares)  # (0, 1, 4, 9, 16)

The generator expression is consumed by tuple().

Common Mistakes

  • Forgetting comma in single-element tuple
  • Assuming parentheses create tuples
  • Trying to modify tuple elements
  • Confusing lists [] with tuples ()

Best Practices

  • Use parentheses for clarity
  • Always include comma for single-element tuples
  • Use tuple() for conversions
  • Use tuples for fixed, constant data

  4. Create a multi-line configuration tuple.

Tuple vs List

Feature Tuple List
Mutable No Yes
Syntax ( ) [ ]
Performance Faster Slightly slower
Use Case Fixed data Changing data

Accessing Tuple Elements

colors = ("red", "green", "blue")

print(colors[0])  # red
print(colors[-1]) # blue

Indexing starts at 0.

Tuple Slicing

numbers = (10, 20, 30, 40, 50)

print(numbers[1:4])  # (20, 30, 40)
tuple[start:end]

Immutability Explained

numbers = (1, 2, 3)
numbers[0] = 10  # TypeError
  • Safer for constant data
  • Memory efficient

Why Use Tuples?

  • Data should not change
  • Returning multiple values from functions
  • Using data as dictionary keys
  • Storing fixed configuration

Tuple Packing & Unpacking

Packing

person = ("Alice", 25, "Engineer")

Unpacking

name, age, job = person

Extended Unpacking

numbers = (1, 2, 3, 4, 5)

first, *middle, last = numbers
  • first = 1
  • middle = [2, 3, 4]
  • last = 5

Tuple Methods

Tuples have only two built-in methods:

numbers = (1, 2, 2, 3)

numbers.count(2)   # 2
numbers.index(3)   # 3

Nested Tuples

matrix = (
    (1, 2),
    (3, 4)
)

print(matrix[0][1])  # 2

Useful for structured data.

Converting Between List and Tuple

Tuple → List

numbers = (1, 2, 3)
numbers_list = list(numbers)

List → Tuple

numbers_tuple = tuple(numbers_list)

Tuples as Dictionary Keys

locations = {
    (40.7128, -74.0060): "New York",
    (34.0522, -118.2437): "Los Angeles"
}

Lists cannot be used as keys.

Performance Benefits

  • Use less memory
  • Are faster to iterate
  • Provide data integrity

When in Doubt:

Use a tuple for fixed data.

Use a list for changing data.
